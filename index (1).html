Setup Guide
CMM Ads Intelligence — multi-user, Google-authenticated, with per-user config and conversation history.

What you need

① Google Client ID   — for user sign-in (see Google Sign-In tab)
② AI API Key         — your AI API key
③ Cloudflare Worker  — proxy for Google Ads + Meta APIs (see Worker tab)
④ Google Ads creds   — developer token + OAuth (see Google Ads tab)
⑤ Meta access token  — system user token (see Meta Ads tab)

Multi-user model

Each person who signs in with Google gets their own private config (API keys, Worker URL, Customer ID, Meta account) and conversation history — stored in their browser's localStorage. No shared backend needed.

1. Create a Google Cloud project

Go to console.cloud.google.com → New Project.

2. Enable Google Identity API

APIs & Services → Enable APIs → search Google Identity → Enable.

3. Create OAuth credentials

APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID → Web application.

Add to Authorised JavaScript origins:
  http://localhost (for local testing)
  https://YOUR-USERNAME.github.io (for GitHub Pages)

4. Copy the Client ID

It looks like 123456789-xxxx.apps.googleusercontent.com. Paste it into Config → Google Client ID.

The Client ID is public — it's safe to have in the HTML. Your actual AI and API credentials are never exposed.

Go to dash.cloudflare.com → Workers & Pages → Create Worker → paste worker.js → Deploy. Then add the 7 secrets in Worker Settings → Variables.

GOOGLE_ADS_DEVELOPER_TOKEN
GOOGLE_ADS_CLIENT_ID
GOOGLE_ADS_CLIENT_SECRET
GOOGLE_ADS_REFRESH_TOKEN
META_ACCESS_TOKEN
META_AD_ACCOUNT_ID
AUTH_SECRET
